# 从一个 C 程序到模拟器执行——完整数据流追踪

> 本文用一个极简 C 程序 `int main() { return 3; }`，逐步骤展示数据形态如何变化。

---

## 第 0 步：你写了一个 C 程序

你在编辑器里写了这样一个文件 `hello.c`：

```c
// hello.c
int main() {
    return 3;
}
```

就三行。这个程序什么都不做，只是向操作系统返回数字 3。

**现在的问题是：这个文本文件怎么变成 CPU 能执行的东西？**

---

## 第 1 步：编译——文本变成汇编

你在终端执行：

```bash
riscv64-linux-gnu-gcc -march=rv32im -mabi=ilp32 -S hello.c
```

产生 `hello.s`（汇编文件）。编译器把 C 语言翻译成了 RISC-V 汇编：

```asm
# hello.s
main:
    addi  x10, x0, 3     # x10 = 0 + 3（x10 是返回值寄存器）
    jalr  x0, x1, 0      # 返回到调用者（x1 存着返回地址）
```

发生了什么：
- `return 3;` 变成了 `addi x10, x0, 3` —— 把 3 放进 x10 寄存器
- 函数的返回变成了 `jalr x0, x1, 0` —— 跳回调用者

**数据形态变化**：

```
之前：文本字符 'i' 'n' 't' ' ' 'm' 'a' 'i' 'n' '(' ')' ...
之后：文本字符 'a' 'd' 'd' 'i' ' ' 'x' '1' '0' ...
```

---

## 第 2 步：汇编——文本变成数字

```bash
riscv64-linux-gnu-gcc -march=rv32im -mabi=ilp32 -c hello.s
```

产生 `hello.o`。汇编器把每行汇编翻译成 **32 位数字**：

```
汇编                        机器码（十六进制）      机器码（二进制）
──────────────────────────────────────────────────────────────────
addi  x10, x0, 3     →     0x00300513           0000 0000 0011 0000 0000 0101 0001 0011
jalr  x0, x1, 0      →     0x00008067           0000 0000 0000 0000 1000 0000 0110 0111
```

**为什么要这样翻译？** 因为 RISC-V 规范规定了 `addi` 指令的格式：

```
addi 的 encoding：
┌──────────────────────┬────────┬─────┬─────────┬─────────┐
│      imm[11:0]       │  rs1   │fuct3│   rd    │ opcode  │
│       3              │   x0   │ 000 │   x10   │ 0010011 │
│  000000000011        │ 00000  │ 000 │  01010  │ 0010011 │
└──────────────────────┴────────┴─────┴─────────┴─────────┘

拼起来：0000 0000 0011 0000 0000 0101 0001 0011 = 0x00300513
```

**数据形态变化**：

```
之前：文本 "addi x10, x0, 3"
之后：32 位数字 0x00300513
```

如果你用十六进制编辑器打开 `hello.o`，你会看到：

```
偏移      数据（十六进制）          解释
0x00:     13 05 30 00              这就是 0x00300513（注意小端序：低字节在前）
0x04:     67 80 00 00              这就是 0x00008067
```

> **小端序说明**：数字 `0x00300513` 在内存中存储为 `13 05 30 00`，即最低字节 `0x13` 存在最低地址，最高字节 `0x00` 存在最高地址。RISC-V 和 x86 都是小端序，所以你们的模拟器通常不需要做端序转换。

## 第 3 步：链接——目标文件变成完整 ELF

`hello.o` 只有你的 `main` 函数。但一个真正能在操作系统上跑的程序，还需要**启动代码**——在 `main` 之前做一些初始化（设置栈、全局变量等）。

链接器会把 `hello.o` 和系统提供的启动文件（`crt0.o` 等）以及 C 库（`libc.a`）合并：

```bash
riscv64-linux-gnu-gcc -march=rv32im -mabi=ilp32 -static -o hello hello.c
```

产生 `hello`（完整的 ELF 可执行文件）。

---

## 第 4 步：ELF 文件里到底有什么

用十六进制编辑器打开 `hello`，我标注每部分是什么：

```
地址        数据（十六进制）                          含义
────────────────────────────────────────────────────────────────────
00000000:  7F 45 4C 46 01 01 01 00 00 00 00 00...   ← ELF Header
           ^^^^^^^^
           Magic Number: 0x7F + "ELF"
           这是 ELF 文件的"身份证"

           ELF Header 告诉我们：
           · 这是 32 位 ELF（01）
           · 小端序（01）
           · RISC-V 机器（e_machine = 0xF3 = 243）
           · 入口地址 = 0x10074（程序的第一条指令在哪）
           · Program Header 表在文件偏移 0x34 处
           · 有 4 个 Program Header

00000034:  01 00 00 00 00 00 00 00...               ← Program Header #1
           ^^
           p_type = 0x00000001 = PT_LOAD
           意思：这一段需要加载到内存里！

           具体内容：
           p_offset  = 0x00000000（从文件开头读数据）
           p_vaddr   = 0x00010000（加载到虚拟地址 0x10000）
           p_filesz  = 0x00000384（文件里 900 字节）
           p_memsz   = 0x00000384（内存里也是 900 字节）
           p_flags   = 0x5（PF_R | PF_X = 可读+可执行 → 这是代码段！）

000000B8:  01 00 00 00 ...                           ← Program Header #2
           p_type   = PT_LOAD
           p_offset = 0x00001000（文件偏移 0x1000）
           p_vaddr  = 0x00012000（虚拟地址 0x12000）
           p_filesz = 0x00000100（文件里 256 字节）
           p_memsz  = 0x00000108（内存里 264 字节 ← 多了 8 字节 .bss！）
           p_flags  = 0x6（PF_R | PF_W = 可读+可写 → 这是数据段！）

           ★ 注意 memsz > filesz：多出来的 8 字节要清零（.bss 段）

00001074:  ...                                       ← 这是 0x1074 偏移处
           这里是 .text 段的实际机器码
           （入口地址 0x10074 指向这里）
```

## 第 5 步：ELF 加载——文件字节拷贝到模拟器内存

你们的模拟器启动时，执行 `rvsim hello`。

模拟器的 ELF 加载器开始工作。它读 ELF 文件，执行以下操作：

```c
// 伪代码：模拟器加载 ELF 的过程

// ① 读取 ELF Header
fread(hello_file, &elf_header, sizeof(elf_header));
// 验证: elf_header.magic == "\x7F ELF"
// 验证: elf_header.machine == 243 (RISC-V)
// 记录: 入口地址 = elf_header.entry = 0x10074

// ② 遍历 Program Headers
for (每个 Program Header) {

    // 处理 Program Header #1:
    //   p_vaddr = 0x10000, p_filesz = 0x384, p_flags = 0x5 (R|X)
    //
    //   在你的模拟器里分配一个 MemRegion：
    sim->mem.regions[0] = {
        .base = 0x10000,
        .size = 0x384,
        .data = malloc(0x384),       // ← 真正分配内存！
        .prot = PROT_READ | PROT_EXEC
    };

    //   从文件读取数据，拷贝到 region->data：
    fseek(hello_file, p_offset, SEEK_SET);  // 定位到文件开头
    fread(region->data, p_filesz, 1, hello_file); // 读 0x384 字节

    // 处理 Program Header #2:
    //   p_vaddr = 0x12000, p_filesz = 0x100, p_memsz = 0x108
    //   p_flags = 0x6 (R|W)
    //
    sim->mem.regions[1] = {
        .base = 0x12000,
        .size = 0x108,               // 注意是 memsz 不是 filesz！
        .data = malloc(0x108),
        .prot = PROT_READ | PROT_WRITE
    };

    //   从文件读 0x100 字节
    fseek(hello_file, 0x1000, SEEK_SET);
    fread(region->data, 0x100, 1, hello_file);
    //   剩余 8 字节 (.bss) 清零
    memset(region->data + 0x100, 0, 8);
}

// ③ 设置栈区域
sim->mem.regions[2] = {
    .base = 0xBFFC0000,
    .size = 0x40000,
    .data = malloc(0x40000),
    .prot = PROT_READ | PROT_WRITE
};

// ④ 设置 CPU 初始状态
sim->cpu.pc      = 0x10074;   // 从 elf_header.entry 来
sim->cpu.regs[2] = 0xC0000000; // sp = 栈顶
sim->cpu.running = true;
```

## 第 6 步：CPU 开始执行

准备好了内存和寄存器，模拟器进入**执行循环**：

```
当前状态:
  PC = 0x10074
  regs[0] = 0, regs[1] = 某值, regs[2] = 0xC0000000, regs[3..31] = 0
```

### 第 1 条指令：`addi x10, x0, 3`

```
① Fetch（取指）:
   PC = 0x10074
   从哪个内存区域？→ Region[0]（基址 0x10000）
   偏移 = 0x10074 - 0x10000 = 0x74
   去 region[0].data[0x74] 读 4 字节
   读到: 13 05 30 00（小端序）
   拼成 32 位: 0x00300513

   汇编含义: addi x10, x0, 3

② Decode（译码）:
   opcode = 0x00300513 & 0x7F = 0x13  → OP-IMM
   rd     = (0x00300513 >> 7) & 0x1F = 10  → x10 (a0)
   funct3 = (0x00300513 >> 12) & 0x7 = 0   → ADDI
   rs1    = (0x00300513 >> 15) & 0x1F = 0  → x0 (zero)
   imm    = (int32_t)(0x00300513 >> 20) = 3

   识别出: ADDI a0, zero, 3

③ Execute（执行）:
   regs[10] = regs[0] + 3 = 0 + 3 = 3

④ Memory（访存）:
   ADDI 不需要访存，跳过

⑤ Writeback（写回）:
   结果 3 已经在 regs[10] 里了

   别忘了！如果 rd == 0，清空：
   if (rd == 0) regs[0] = 0;  // rd=10，不用管

更新 PC:
   PC = PC + 4 = 0x10078

当前状态:
  PC = 0x10078
  regs[10] = 3  ← main 的返回值设置好了！
  其他不变
```

### 第 2 条指令：`jalr x0, x1, 0`

```
① Fetch:
   PC = 0x10078 → Region[0] 偏移 0x78
   读到: 67 80 00 00 → 0x00008067

   汇编含义: jalr x0, x1, 0

② Decode:
   opcode = 0x67 → JALR
   rd  = 0  → x0（返回值不用存）
   rs1 = 1  → x1（ra，返回地址）
   imm = 0

③ Execute:
   计算目标地址 = (regs[1] + 0) & ~1 = (某值) & ~1

   比如 regs[1] 里面存的是调用 main 的返回地址，假设是 0x10A00
   那 target = 0x10A00

   注意 rd=0，返回值丢弃（符合预期，因为这是 main 返回）

更新 PC:
   PC = target = 0x10A00  ← 跳回去了！

...

接下来可能跑到 _exit(3)：
  会设置 a7=93, a0=3, 然后 ecall
```

## 第 7 步：遇到 ECALL——系统调用

继续执行若干条指令后……

```
═══════════════════════════════════════════════
某条指令: ECALL
═══════════════════════════════════════════════

① Fetch:
   PC = 某地址 → 读到: 73 00 00 00 → 0x00000073

② Decode:
   opcode = 0x73 → SYSTEM
   这是 ECALL！

③ Execute:
   syscall_handler(sim);

   当前寄存器:
   a7 (x17) = 93   → syscall 号 = exit
   a0 (x10) = 3    → 参数 = 退出码

   模拟器执行:
   case 93: // exit
       sim->cpu.running = false;   // 停止模拟！
       printf("Program exited with code %d\n", regs[10]);
       // 输出: Program exited with code 3
       break;

═══════════════════════════════════════════════
模拟结束
═══════════════════════════════════════════════
```

### 两种视角的对应关系

```
编译时（工具链做的事）              运行时（你们模拟器做的事）
═══════════════════════          ═════════════════════════
                                                                 
hello.c                           你们的模拟器不管是 .c 还是 .s，
  ↓ gcc -S                        只管加载最终的 ELF 文件。
hello.s                                                          
  ↓ gcc -c                        ELF 加载器                      
hello.o                           解析 ELF Header + Program Headers
  ↓ gcc -static -o                把段数据拷贝到虚拟内存              
hello (ELF)                                                    
                                  CPU 执行循环                    
  你们不用管编译过程，              取指 → 译码 → 执行 → 访存 → 写回
  只需要加载最终的 ELF！           遇到 ecall → 系统调用处理          
                                  遇到 ebreak → 调试器交互          
```

---

> **一句话总结**：`int main() { return 3; }` 这个文本，经过编译→汇编→链接变成 ELF 文件，模拟器把 ELF 里的代码和数据拷进虚拟内存，然后逐条取指、译码、执行，最终 `exit(3)` 停止模拟。调试器让你在这个过程中随时暂停、查看、单步前进。
