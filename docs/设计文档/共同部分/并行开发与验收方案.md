> 目标：四人全线并行推进，不等不靠。统一接口（头文件 = 合约），各自用 Stub 独立开发测试，最后替换 Stub 一步联调。

```

         │         types.h（零依赖）              │
         │   PrivilegeLevel, ExceptionType,      │

         ┌──────────┴───────────────────────────┐

         │   loader.h  │  debugger.h            │     Day 0 冻结
```

```c
#ifndef TYPES_H
#define TYPES_H

#include <stdint.h>
#include <stdbool.h>
#include <stddef.h>

/* ── 特权级 ── */
typedef enum {
    PRIV_USER       = 0,
    PRIV_SUPERVISOR = 1,
    PRIV_MACHINE    = 3,
} PrivilegeLevel;

/* ── 异常类型 ── */
typedef enum {
    EXC_NONE                   = 0,
    EXC_INST_ADDR_MISALIGNED   = 1,
    EXC_INST_ACCESS_FAULT      = 2,
    EXC_ILLEGAL_INST           = 3,
    EXC_BREAKPOINT             = 4,
    EXC_LOAD_ADDR_MISALIGNED   = 5,
    EXC_LOAD_ACCESS_FAULT      = 6,
    EXC_STORE_ADDR_MISALIGNED  = 7,
    EXC_STORE_ACCESS_FAULT     = 8,
    EXC_ECALL_U                = 9,
    EXC_ECALL_S                = 10,
    EXC_ECALL_M                = 11,
    EXC_PAGE_FAULT_INST        = 12,
    EXC_PAGE_FAULT_LOAD        = 13,
    EXC_PAGE_FAULT_STORE       = 15,
} ExceptionType;

/* ── 内存权限标志 ── */
#define MEM_READ   (1 << 0)
#define MEM_WRITE  (1 << 1)
#define MEM_EXEC   (1 << 2)

/* ── PTE 权限标志 ── */
#define PTE_VALID    (1 << 0)
#define PTE_READ     (1 << 1)
#define PTE_WRITE    (1 << 2)
#define PTE_EXEC     (1 << 3)
#define PTE_USER     (1 << 4)
#define PTE_GLOBAL   (1 << 5)
#define PTE_ACCESSED (1 << 6)
#define PTE_DIRTY    (1 << 7)

/* ── Sv32 常量 ── */
#define PAGE_SIZE    4096
#define PAGE_SHIFT   12
#define SATP_MODE_OFF  0
#define SATP_MODE_SV32 1

/* ── MEM → PTE 权限转换 ── */
static inline uint8_t mem_perm_to_pte_flags(uint8_t mem_flags) {
    uint8_t pte = PTE_VALID;
    if (mem_flags & MEM_READ)  pte |= PTE_READ;
    if (mem_flags & MEM_WRITE) pte |= PTE_WRITE;
    if (mem_flags & MEM_EXEC)  pte |= PTE_EXEC;
    return pte;
}

#endif // TYPES_H
```

## 阶段 1：共享 Stub 库 —— 解耦的魔法

Stub 是"签名与真实模块一致、实现极简但功能足够测试"的假模块。四人用同一套 Stub，各自测试自己的真实模块。

### Stub 设计总览

```
src/stub/
├── stub_memory.h          ← 直接用真实 memory.h 的 PhysicalMemory 实现
├── stub_memory.c          ← 参考项目1 memory.c 的实现（已有，搬过来即可）
├── stub_mmu.h             ← Bare 模式 MMU（vaddr = paddr，不走页表）
├── stub_mmu.c             ← 约 80 行：mmu_init + 恒等映射的 read/write
├── stub_cpu.h             ← 提供 CPU 区段的 fake 状态 + cpu_disasm
├── stub_cpu.c             ← 约 50 行：fake regs[] + pc + running
├── stub_simulator.h       ← 把 stub 各模块串成 Simulator
├── stub_simulator.c       ← 约 60 行：sim_init + sim_step + sim_run
└── README.md              ← 使用说明
```

**关键设计决策：Stub 不引入新类型。** Stub 使用和真实头文件完全相同的 `PhysicalMemory`、`MMUState`、`CPU`、`Simulator` 类型。替换时只需改 Makefile 链接目标，无需改任何 `#include`。

### 1.1 stub_memory.c（就是真实 PhysicalMemory 的完整实现）

因为 PhysicalMemory 零依赖（只依赖 `types.h` + `stdlib.h`），**Stub 即真实实现**。焕聪写的 `memory.c` 本身就是 Stub。这意味着：

- 焕聪的 `memory.c` → 放在 `src/src/memory/memory.c`（真实模块）
- 其他人的 Stub → 直接链接焕聪的 `memory.o`，或者自己维护一份副本

**最简方案：焕聪先提交 `memory.c`（纯 PhysicalMemory，不涉及 MMU），其他人直接链接它。** PhysicalMemory 是纯数据操作（数组读写 + region 管理），第一版写对后很少变动。

### 1.2 stub_mmu.c —— Bare 模式 MMU

```c
// stub_mmu.c — Bare 模式 MMU（vaddr = paddr，不走页表）
// 接口签名与真实的 mmu.h 完全一致
// 李特/嘉华/嘉俊 用这个独立测试自己的模块

#include "types.h"
#include "memory/memory.h"
#include "memory/mmu.h"
#include <stdlib.h>
#include <string.h>

void mmu_init(MMUState *mmu) {
    mmu->satp = 0;
    mmu->enabled = false;
    mmu->root_page_table = NULL;  // Bare 模式不需要页表
}

bool mmu_translate(MMUState *mmu, uint32_t vaddr, uint32_t *paddr,
                   bool is_write, bool is_exec, PrivilegeLevel priv,
                   ExceptionType *exc) {
    (void)mmu; (void)is_write; (void)is_exec; (void)priv;
    *paddr = vaddr;   // Bare 模式：vaddr = paddr
    *exc = EXC_NONE;
    return true;
}

// 以下函数模式相同：translate → mem_read/write
bool mmu_read_8(MMUState *mmu, PhysicalMemory *pmem, uint32_t vaddr,
                uint8_t *val, PrivilegeLevel priv, ExceptionType *exc) {
    uint32_t paddr;
    if (!mmu_translate(mmu, vaddr, &paddr, false, false, priv, exc))
        return false;
    bool ok = mem_read_8(pmem, paddr, val);
    if (!ok && exc) *exc = EXC_LOAD_ACCESS_FAULT;
    return ok;
}

        return false;
    bool ok = mem_write_8(pmem, paddr, val);
    if (!ok && exc) *exc = EXC_STORE_ACCESS_FAULT;
    return ok;
}

bool mmu_write_16(MMUState *mmu, PhysicalMemory *pmem, uint32_t vaddr,
                  uint16_t val, PrivilegeLevel priv, ExceptionType *exc) {
    uint32_t paddr;

    if (!ok && exc) *exc = EXC_STORE_ACCESS_FAULT;
    return ok;
}

bool mmu_write_32(MMUState *mmu, PhysicalMemory *pmem, uint32_t vaddr,
                  uint32_t val, PrivilegeLevel priv, ExceptionType *exc) {
    uint32_t paddr;
    if (!mmu_translate(mmu, vaddr, &paddr, true, false, priv, exc))
        return false;
    bool ok = mem_write_32(pmem, paddr, val);
    if (!ok && exc) *exc = EXC_STORE_ACCESS_FAULT;
    return ok;
}

// 批量读写：逐字节循环
bool mmu_read(MMUState *mmu, PhysicalMemory *pmem, uint32_t vaddr,
              uint8_t *buf, uint32_t len, PrivilegeLevel priv,
              ExceptionType *exc) {
    for (uint32_t i = 0; i < len; i++) {
        if (!mmu_read_8(mmu, pmem, vaddr + i, &buf[i], priv, exc))
            return false;
    }
    *exc = EXC_NONE;
    return true;
}

bool mmu_write(MMUState *mmu, PhysicalMemory *pmem, uint32_t vaddr,
               const uint8_t *buf, uint32_t len, PrivilegeLevel priv,
               ExceptionType *exc) {
    for (uint32_t i = 0; i < len; i++) {
        if (!mmu_write_8(mmu, pmem, vaddr + i, buf[i], priv, exc))
            return false;
    }
    *exc = EXC_NONE;
    return true;
}

// Stub 的 mmu_map_page：不做实际页表操作，直接返回成功
bool mmu_map_page(MMUState *mmu, uint32_t vaddr, uint32_t paddr,
                  uint8_t flags) {
    (void)mmu; (void)vaddr; (void)paddr; (void)flags;
    return true;
}

// Stub 的 mmu_brk：转发到 PhysicalMemory
uint32_t mmu_brk(MMUState *mmu, PhysicalMemory *pmem, uint32_t new_brk) {
    (void)mmu;
    return mem_brk(pmem, new_brk);
```

```c
    "t0", "t1", "t2",
    "s0", "s1",
    "a0", "a1", "a2", "a3", "a4", "a5", "a6", "a7",
    "s2", "s3", "s4", "s5", "s6", "s7", "s8", "s9", "s10", "s11",
    "t3", "t4", "t5", "t6"
};

void cpu_init(CPU *cpu) {
    memset(cpu, 0, sizeof(CPU));
    cpu->priv = PRIV_MACHINE;
}

void cpu_reset(CPU *cpu) {
    cpu_init(cpu);
}

// Stub 版的 cpu_disasm：简单实现，供 Debugger 测试
void cpu_disasm(uint32_t instr, uint32_t pc, char *buf, size_t bufsz) {
    uint8_t opcode = instr & 0x7F;
    uint8_t rd     = (instr >> 7) & 0x1F;
    uint8_t rs1    = (instr >> 15) & 0x1F;
    int32_t imm_i  = (int32_t)(instr >> 20);

    switch (opcode) {
    case 0x13:  // ADDI
        snprintf(buf, bufsz, "addi %s, %s, %d",
                 reg_names[rd], reg_names[rs1], imm_i);
        break;
    case 0x33:  // ADD
        snprintf(buf, bufsz, "add %s, %s, %s",
                 reg_names[rd], reg_names[rs1], reg_names[(instr >> 20) & 0x1F]);
        break;
    case 0x73:  // ECALL / EBREAK
        if (imm_i == 0) snprintf(buf, bufsz, "ecall");
        else if (imm_i == 1) snprintf(buf, bufsz, "ebreak");
        else snprintf(buf, bufsz, "unknown");
        break;
    case 0x03:  // LW
        snprintf(buf, bufsz, "lw %s, %d(%s)",
                 reg_names[rd], imm_i, reg_names[rs1]);
        break;
    case 0x23:  // SW
        snprintf(buf, bufsz, "sw %s, %d(%s)",
                 reg_names[(instr >> 20) & 0x1F], IMM_S(instr), reg_names[rs1]);
        break;
    default:
        snprintf(buf, bufsz, "?");
        break;
    }
}
```

### 1.4 stub_simulator.c —— 串联所有 Stub

```c
// stub_simulator.c — 用 Stub 组件拼出一个可跑的 Simulator
// 用于：李特测试 CPU 执行、嘉俊测试 Debugger REPL
// 等所有真实模块就绪后，替换为真实的 sim.o

#include "simulator.h"
#include "cpu/cpu.h"
#include "cpu/decode.h"
#include "cpu/execute.h"
#include <stdio.h>
#include <stdlib.h>

void sim_init(Simulator *sim) {
    memset(sim, 0, sizeof(*sim));
    cpu_init(&sim->cpu);
    mmu_init(&sim->mmu);
    mem_init(&sim->pmem, MEM_SIZE_DEFAULT);
    sim->bp_capacity = 16;
    sim->breakpoints = (Breakpoint *)calloc(16, sizeof(Breakpoint));
}

void sim_destroy(Simulator *sim) {
    mem_destroy(&sim->pmem);
    free(sim->breakpoints);
    sim->breakpoints = NULL;
}

// 单步执行（Stub 版：fetch + decode + execute 一条龙）
void sim_step(Simulator *sim) {
    ExceptionType exc = EXC_NONE;
    uint32_t Instr;
    if (!mmu_read_32(&sim->mmu, &sim->pmem, sim->cpu.pc,
                     &Instr, sim->cpu.priv, &exc)) {
        printf("Fetch fault at 0x%08x\n", sim->cpu.pc);
        sim->cpu.running = false;
        return;
    }

    DecodedInstr d = cpu_decode(Instr);
    uint32_t next_pc = sim->cpu.pc + 4;

    if (!cpu_execute(sim, &d, &next_pc)) {
        printf("Execute fault at 0x%08x\n", sim->cpu.pc);
        sim->cpu.running = false;
        return;
    }

    sim->cpu.pc = next_pc;
    sim->cpu.regs[0] = 0;
    sim->inst_count++;
    sim->cycle_count++;

    if (sim->single_step) {
        sim->cpu.running = false;
        sim->single_step = false;
    }
}

void sim_run(Simulator *sim) {
    sim->cpu.running = true;
    while (sim->cpu.running) {
        sim_step(sim);
    }
}

// sim_load_elf 的 Stub 版：嘉华独立测试时用真正的 elf_load，
// 这里提供空壳给不需要 Loader 的测试场景
bool sim_load_elf(Simulator *sim, const char *path) {
    (void)sim; (void)path;
    return true;  // no-op stub
}
```

```c
#include "cpu/decode.h"
#include "cpu/execute.h"

#include <assert.h>

// 初始化 Simulator 并加载指令到内存

    sim->cpu.priv = PRIV_MACHINE;

// 把指令数组写入内存（从 addr 开始）
static inline void test_load_Instrs(Simulator *sim, uint32_t addr,
                                    const uint32_t *Instrs, int count) {
    ExceptionType exc;
    for (int i = 0; i < count; i++) {
        mmu_write_32(&sim->mmu, &sim->pmem, addr + i * 4,
                     Instrs[i], PRIV_MACHINE, &exc);

        sim_step(sim);
    }
}

        return 1; \

} while(0)

#define CHECK_EQ(got, expected, msg) do { \
    if ((got) != (expected)) { \
        fprintf(stderr, "FAIL: %s: got %d, expected %d\n", \
```

## 阶段 3：集成 —— 替换 Stub，逐步联调

当某个模块的真实实现通过了自己的全部测试后，就可以替换对应的 Stub。

**集成顺序**：

```
Step 1: 焕聪 memory.c 就绪 → 其他人把 Stub 的 memory.o 替换为真实的
        （几乎无感知，因为 Stub 的 PhysicalMemory 就是真实实现）

Step 2: 焕聪 mmu.c 就绪（含 Sv32）→ 李特把 stub_mmu.o 替换为真实 mmu.o
        → 跑 CPU 全部测试，确认所有 case 仍然通过
        → 如果有失败：说明之前 Stub MMU 的行为和真实 MMU 有差异，定位修复

Step 3: 嘉华 elf_load 就绪 → 李特用 sim_load_elf 替换 test_load_Instrs
        → 跑端到端：ELF → Loader → MMU → CPU → exit
        → 跑 hello.s（需要交叉编译器产出 ELF）

Step 4: 李特 CPU 全部阶段完成 → 嘉俊把 stub_cpu.o 替换为真实 cpu.o
        → 确认 Debugger 的寄存器显示、x 命令、断点命中全部正常

Step 5: 嘉俊 Debugger 全部命令完成 → 全员联调
        → main.c 串联所有真实模块
```

## 常见问题

**Q: Stub 的行为和真实模块不一致，导致我的测试通过了但联调时出错怎么办？**

A: 这正是 Stub 方案的优势——问题在联调时暴露，但你的模块逻辑是正确且独立验证过的。定位时只需检查"Stub 和真实模块在哪个行为上有差异"，而不是"我的代码哪里写错了"。通常差异出在边界条件（越界、权限、跨页），修复很快。

**Q: 焕聪的 Memory 还没写好，我能开始吗？**

A: 能。李特用 `stub_mmu.c`（Bare 模式 vaddr=paddr），嘉华用 `stub_mmu.c`（`mmu_map_page` 是 no-op），嘉俊用 `stub_mmu.c` + `stub_cpu.c`。所有人第一天就能写代码。

**Q: 如果接口签名需要改怎么办？**

A: 先改头文件（合约），然后立即通知所有人。Stub 和真实实现同步更新。因为四人都在同一个 repo 的不同目录下工作，改头文件后各自 `make test` 就能发现编译错误，5 分钟内修完。

**Q: 测试 ELF 文件从哪来？**

A: 嘉华写一个 C 程序（跑在 x86 宿主机上）生成最小 RISC-V ELF32 文件，不需要交叉编译器。等交叉编译环境搭好后，再用 GCC 生成更复杂的测试用例。

## 第二阶段目标

用 `riscv64-unknown-elf-gcc` 编译的 C 程序，跑在模拟器上，`printf("Hello, World!")` 能正常输出。

## 差距分析与待办（按优先级）

### P0：Syscall 补齐（阻塞所有 C 程序运行）

一个 `printf("hello")` 的 C 程序编译后，实际调用的 syscall：

| syscall | 编号 (a7) | 作用 | 状态 | 难度 |
|---------|-----------|------|------|------|
| `exit` | 93 | 程序退出 | ✅ 已有 | — |
| `write` | 64 | 输出到终端 | ✅ 已有 | — |
| `brk` | 214 | 堆空间管理（malloc 依赖） | ❌ 待补 | ⭐⭐ |
| `read` | 63 | 标准输入（scanf 依赖） | ❌ 待补 | ⭐ |
| `openat` | 56 | 打开文件（fopen 依赖） | ❌ 待补 | ⭐⭐ |
| `fstat` | 80 | 文件信息 | ❌ 待补 | ⭐ |
| `close` | 57 | 关闭文件 | ❌ 待补 | ⭐ |
| `lseek` | 62 | 文件偏移 | ❌ 待补 | ⭐ |

**P0 最小集合（解锁 Hello World）**：补上 `brk` 即可。`printf` 只需要 `brk`（初始化 malloc）+ `write`（输出）。其他 syscall 按需补。

### P1：C 运行时支持（编译器产出的 ELF 入口是 _start，不是 main）

```
编译器产出 ELF 的典型段:
  .init    — crt0 入口 (_start)
  .text    — main() 等用户代码
  .rodata  — 只读数据（"Hello, World!" 字符串）
  .data    — 已初始化全局变量
  .bss     — 未初始化全局变量（需运行时清零）
```

你的 Loader 现状：
- ✅ PT_LOAD 段映射
- ✅ `.text` / `.rodata` / `.data` 加载
- ✅ `.bss` 清零已有代码（`elf_segment.c` 中 calloc 了 `memsz - filesz` 部分）
- ❌ 栈地址 0x07F00000 与 Linux RISC-V 约定（~0xC0000000）不一致，C runtime 可能不适配
- ❌ 缺少 ELF 辅助向量（aux vector），crt0 初始化时需要

**待办**：
- 统一栈地址到 0xC0000000（`loader_internal.h` vs `simulator.h` 目前有两个不同定义）
- 可选：在 Loader 中构造 aux vector（最少需要 AT_PAGESZ、AT_PHNUM 等）

### P2：启用 Sv32 页表 + 用户态（U-mode）

当前所有程序跑在 M-mode，vaddr = paddr。要让 gcc 编译的程序正确运行：

- Loader 映射段时同步建立页表（`mmu_map_page` 目前是 stub）
- 入口从 M-mode 切换到 U-mode（需要 `mret` 指令，已有基础实现）
- ecall 从 U-mode 进入 M-mode 后，syscall handler 处理完需要 `mret` 返回
- 当前 syscall handler 只是简单 `running = true` + `mcause = 0`，不支持真正的异常返回流程

**待办**：
- 补完 `mmu_translate` 中 PPN → C 指针的页表管理器（约 30 行）
- 补完 `mmu_map_page`（约 20 行）
- syscall handler 从"直接改 running"改为"走完整的 mret 返回路径"

### P3：浮点 + 其他扩展

- RV32F（单精度浮点）：`fadd.s / fsub.s / fmul.s / fdiv.s / flw / fsw` 等 ~26 条指令
- CSR 指令：`csrrw / csrrs / csrrc / csrrwi / csrrsi / csrrci`（用户态 ecall 返回需要）
- 中断：mtvec 不只是异常入口，也支持中断（需要 mstatus.MIE + mie/mip）
