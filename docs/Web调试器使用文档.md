# RISC-V Simulator — Web 调试器使用文档

## 简介

RISC-V Simulator 是一个 RV32IMF 指令集模拟器，内置 Web 可视化调试器。
通过浏览器即可完成加载程序、单步执行、查看数据通路、管理断点、查看内存等调试操作。

## 快速启动

```powershell
# 编译
mingw32-make

# 启动 Web 调试器
./riscv-sim -w 8080 <elf_file>

# 浏览器打开
http://localhost:8080
```

| 参数 | 说明 |
|------|------|
| `-w <port>` | 启动 Web 调试器，监听指定端口 |
| `-s` | 命令行调试模式（不使用 Web） |
| `-t` | 指令跟踪模式 |

## 界面总览

```
┌─ 工具栏 ──────────────────────────────────────────────────┐
│ RISC-V Simulator          [C Compiler] [Stopped] [Step] [Run] [⏻] │
├──────────┬──────────────────────────┬─────────────────────┤
│ Assembly │     Pipeline 数据通路图   │  Registers          │
│ (指令列表) │    (动态信号值实时刷新)   │  CSR                │
│          │                          │  Breakpoints        │
│          │  当前指令详情              │  Memory             │
│          │  PC / 机器码 / 反汇编      │  (Hex 查看器)        │
├──────────┴──────────────────────────┴─────────────────────┤
│ Log (操作日志)                                              │
└──────────────────────────────────────────────────────────┘
```

页面分为四大区域：

| 区域 | 位置 | 功能 |
|------|------|------|
| **工具栏** | 顶部 | 模拟器控制：Step（单步）、Run（全速）、C Compiler（在线编译）、⏻（关闭服务器） |
| **左侧栏** | 左 | 汇编指令列表，当前指令蓝色高亮，点击行可添加/删除断点 |
| **中间** | 中 | 流水线数据通路图（SVG），图上信号值随指令执行实时刷新 |
| **右侧栏** | 右 | 寄存器、CSR、断点列表、Memory 查看器 |

## 核心操作

### 单步执行（Step）

点击 **Step** 按钮，每点击一次执行一条指令。执行后：

- 左侧 Assembly 列表移动高亮到下一行
- 中间 Pipeline 图上的所有信号值（PC、INST、ALU_A、ALU_B、ALU_OUT、WB 等）同步刷新
- 右侧寄存器值同步更新
- Memory 面板同步刷新

### 全速运行（Run）

点击 **Run** 按钮，程序全速执行直到：
- 遇到断点（Breakpoint）
- 程序正常退出（ecall exit）
- 发生异常

**注意**：如果没有设置断点，程序会瞬间跑完（毫秒级），看起来像"没反应"，实际上是立刻结束了。建议先设断点再 Run。

### 断点（Breakpoint）

**添加断点**（两种方式）：
1. **点击汇编行**：在左侧 Assembly 列表中直接点击某条指令 → 自动添加断点，该行出现 ● 红点
2. **手动输入**：在右侧 Add BP 输入框输入地址 → 点 `+`

**删除断点**：
1. 再次点击已有断点的汇编行
2. 在右侧 Breakpoints 列表中点 `Del` 按钮

**断点工作原理**：设置断点后点 Run，程序全速执行到断点地址停下，所有面板刷新到断点处的状态。

### 在线编译（C Compiler）

点击工具栏 **C Compiler** 按钮进入在线编译页面：

1. 左侧编辑器输入 C 代码（或选择 Sample 示例）
2. 按 `Ctrl+Enter` 或点击 **Compile & Load** 编译
3. 编译成功 → 出现蓝色 **Open Debugger →** 按钮 → 点击回到调试器
4. 程序已自动加载，可直接 Step 调试

**支持的 C 语法**：基本运算、函数调用、循环、数组、浮点运算。编译目标为 `rv32imf`，不支持标准库。

## 面板详解

### 流水线数据通路图

Pipeline 图是一个 RV32 五级流水线（IF → ID → EX → MEM → WB）的静态结构图，
上面叠加了 **18 个动态信号标签**，显示当前指令的实际硬件信号值：

| 标签 | 含义 | 颜色 |
|------|------|------|
| PC / INST / DISASM | 当前指令地址、机器码、反汇编 | 蓝色（指令流） |
| RS1 / RS2 / IMM | 源寄存器值、立即数 | 蓝色 |
| ALU_A / ALU_B / OP / ALU_OUT | ALU 输入、操作、输出 | 蓝/绿/橙 |
| MEM_ADDR / MEM_RD / MEM_WD | 内存地址、读数据、写数据 | 橙色（访存） |
| WB | 写回目的寄存器和值 | 绿色 |
| NPC / FWD / STALL / FLUSH | 下一 PC、转发、停顿、冲刷 | 灰/红（控制） |

每点一次 Step，所有标签值同步刷新。

### 寄存器（Registers）

显示 32 个通用寄存器（x0-x31）的当前值。每行两个寄存器，格式为 `x号 ABI名 十六进制值`。

### CSR（控制状态寄存器）

显示 6 个关键 CSR：

| 寄存器 | 含义 |
|--------|------|
| pc | 程序计数器 |
| mstatus | 机器状态 |
| mtvec | 异常入口地址 |
| mepc | 异常返回地址 |
| mcause | 异常原因 |
| mtval | 异常附带信息 |

### Memory（内存查看器）

```
┌─ Memory ────────────────────┐
│ [.text] [.stack]             │  ← 快捷跳转
│ ◀ [0x10000    ] ▶ Go       │  ← 地址导航
│ ─────────────────────────── │
│ 00010000: 04000893 ....    │  ← Hex 内容
│ 00010010: 00E00613 ....    │
└─────────────────────────────┘
```

| 操作 | 功能 |
|------|------|
| `.text` | 跳到当前 PC（查看代码段机器码） |
| `.stack` | 跳到栈指针 SP 当前地址（查看栈数据） |
| `◀ ▶` | 前后翻页 64 字节 |
| 输入地址 + Enter / Go | 跳转任意地址 |
| **自动刷新** | 每次 Step 后自动刷新当前查看区域 |

## 演示脚本（~5 分钟）

### 准备

```powershell
./riscv-sim -w 8080 src/test/e2e/hello.elf
# 浏览器打开 http://localhost:8080
```

### 演示流程

**1. 数据通路可视化（30 秒）**
- 指着 Pipeline 图介绍："这是 RV32 五级流水线，蓝色标签是真实信号值"
- 点一次 Step，展示图上数值变化

**2. 单步调试（1 分钟）**
- 连续点 3-4 次 Step
- 每次观察：左侧汇编高亮移动 → 中间信号刷新 → 右侧寄存器变化
- 重点展示：`addi a7, zero, 64` 执行后 x17 变为 0x40

**3. C 在线编译（1 分钟）**
- 点 **C Compiler** → 选 Fibonacci Sample → **Compile & Load**
- 编译成功 → 点 **Open Debugger** 回到调试器
- 展示刚刚编译的 fib 程序

**4. 断点 + Run（30 秒）**
- 在 Assembly 中点击 fib 函数的某条指令（加断点 ●）
- 点 **Run** → 程序全速跑到断点停下
- 展示断点处数据通路状态

**5. Memory 查看器（30 秒）**
- 点 `.stack` 跳栈空间，Step 一下看栈变化
- 点 `.text` 跳代码段，展示机器码和反汇编的对应关系

### 演示要点

| 亮点 | 一句话 |
|------|--------|
| 可视化 | "图上每个信号值都是实时从模拟器读取的真实状态" |
| 在线编译 | "C 代码一键编译成 RISC-V ELF，立即调试" |
| 全栈自研 | "编译器、TCP 服务器、CPU 模拟器全部手写" |
